# ローカル成果物の保管

ハーネスが生成する一時ファイルは、機能ごとに `.scratch/<機能名>/` へ保存する。このディレクトリには、ローカルの仕様・チケット、テスト実行結果、スクリーンショット、ログ、調査メモ、試作物を置ける。アプリケーションの実行時データは `.scratch/<機能名>/data/` に置く。

品質評価は次の場所に保存する。

```text
.scratch/<機能名>/
├── test-results/             # テストツールの生データ、失敗時の証跡
└── quality/
    ├── test-summary.md       # 実行コマンドごとの結果と最終判定
    └── code-review.md        # 固定点、サブエージェント、標準・仕様レビュー結果
```

`test-summary.md` と `code-review.md` は、実行日時、判定、対象を明記する。サブエージェントを使うレビューでは、役割・対象・完了状態も `code-review.md` に記録する。

`.scratch/` は Git 管理しない。成果物を共有・恒久化する必要が生じた場合は、内容を整理して `docs/`、ソースコード、または GitHub Issue に明示的に移す。

再現に必要なテストコードは Git 管理する。一方、`coverage/`、`test-results/`、`playwright-report/`、`node_modules/` のような再生成可能な出力は Git 管理しない。ルート直下の `.data/` も誤って生成された場合に備えてGit管理外とするが、新規の実行時データの保存先には使わない。
