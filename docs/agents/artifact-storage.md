# ローカル成果物の保管

ハーネスが生成する一時ファイルは、機能ごとに1つだけ作る `.scratch/<機能名>/` へ保存する。同じ機能について `-prototype`、`-test` などの兄弟ディレクトリを作らない。このディレクトリには、ローカルの仕様・チケット、テスト実行結果、スクリーンショット、ログ、調査メモ、試作物を置ける。アプリケーションの実行時データは `.scratch/<機能名>/data/` に置く。

品質評価は次の場所に保存する。

```text
.scratch/<機能名>/
├── spec.md                   # 仕様書
├── tickets/                  # 実装チケット
├── prototype/                # 試作のアプリコード・テストコード
├── data/                     # 試作の実行時データ
├── test-results/             # テストツールの生データ、失敗時の証跡
└── quality/
    ├── test-plan.md          # 合意済みの継ぎ目と振る舞い別テストケース
    ├── tdd-log.md            # 振る舞いごとのRed/Green実行記録
    ├── test-summary.md       # 実行コマンドごとの結果と最終判定
    └── code-review.md        # 固定点、サブエージェント、標準・仕様レビュー結果
```

`test-plan.md` は継ぎ目とテストケースを分けて記録する。`tdd-log.md` は各振る舞いのRed/Greenを記録する。`test-summary.md` と `code-review.md` は、実行日時、判定、対象を明記する。サブエージェントを使うレビューでは、役割・対象・完了状態も `code-review.md` に記録する。

`.scratch/` は Git 管理しない。成果物を共有・恒久化する必要が生じた場合は、内容を整理して `docs/` またはソースコードに明示的に移す。

再現に必要なテストコードは Git 管理する。一方、`coverage/`、`test-results/`、`playwright-report/`、`node_modules/` のような再生成可能な出力は Git 管理しない。ルート直下の `.data/` も誤って生成された場合に備えてGit管理外とするが、新規の実行時データの保存先には使わない。
