# 質問と文書化

プロダクトや設計の曖昧さを解消し、実装前に共有できる判断へ変換する。このスキルは外部スキルを呼び出さない自己完結型である。

## 進め方

1. `AGENTS.md`、`CONTEXT.md`、対象領域の `docs/adr/` を読める場合は読む。
2. 利用者、解決したい問題、成功条件、利用場面、制約、対象外、既存設計との整合を一度に詰め込みすぎず質問する。
3. 回答のたびに、確定事項・未確定事項・判断が必要な選択肢を短く整理して示す。
4. 用語の意味や避ける呼び方が確定したら、`CONTEXT.md` に追記する。用語集がなければ作成する。
5. 後から見直す価値のある技術・運用判断は、`docs/adr/NNNN-<slug>.md` に「背景」「決定」「理由」「影響」を日本語で記録する。判断が確定するまで ADR を作らない。
6. 実装へ進む前に、テストする公開境界（利用者が操作する API、画面、CLI など）を提案し、利用者へ確認する。
7. 要件が十分に固まったら、確定事項と未確定事項を要約し、`/to-spec` を案内する。

## 守ること

- 質問は仮説を検証するために行い、利用者の代わりに重要な要件を決めない。
- 用語と判断は会話だけに残さず、再利用できる文書に残す。
- `grilling` はこのMVPには含めない。深掘り質問が必要な場合も、このスキル内で目的に必要な範囲にとどめる。
